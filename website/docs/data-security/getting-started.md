# Data Security

This page provides a structured deployment approach for securing data across Microsoft 365 using Microsoft Purview, grounded in the **Secure by Default** model.

Start with a protection-first posture — automatically applying sensitivity labels and security controls to reduce oversharing risk — then scale through automation and policy enforcement across your environment. [Learn more about Secure by Default →](https://learn.microsoft.com/purview/deploymentmodels/depmod-secure-by-default-intro)

From this foundation, partners can accelerate implementation with deployment tools and assets designed to operationalize best practices and streamline customer delivery.

## What this enables

- Protect sensitive data across **Microsoft 365 workloads** — Exchange, SharePoint, OneDrive, Teams, and devices.
- Apply **default protection at scale** using sensitivity labels, auto-labeling, and site-based inheritance.
- Reduce risk by **preventing oversharing by default**, with users managing exceptions instead of enforcing protection manually. [Learn more →](https://learn.microsoft.com/purview/deploymentmodels/depmod-secure-by-default-intro)

## Core features

**Microsoft Information Protection (MIP)** — Classify and protect data through sensitivity labels, encryption, and labeling policies that define how data is handled across the environment.

**Data Loss Prevention (DLP)** — Detect, monitor, and control how sensitive data is shared by enforcing policies across Microsoft 365 services and endpoints. [Learn more →](https://learn.microsoft.com/purview/dlp-learn-about-dlp)

Together, MIP and DLP create a **layered data security model** — labels define sensitivity, and policies enforce protection.
