# Data Security

:::tip What's different here?
Microsoft Learn covers MIP and DLP feature docs. This page gives you the **partner deployment workflow** — interactive tools that generate SOWs, design label taxonomies, automate policy rollout, and industry one-pagers you can hand directly to customers.
:::

Protect sensitive data across Microsoft 365 workloads — email, SharePoint, OneDrive, Teams, and endpoints.

Data Security for Business Premium covers two core areas: **Microsoft Information Protection (MIP)** for classifying and protecting data, and **Data Loss Prevention (DLP)** for controlling how data is shared.

## Choose your starting point

| Area | What it covers | Status |
| --- | --- | --- |
| [Getting Started with MIP](./mip.md) | Sensitivity labels, encryption, auto-labeling, visual markings, and industry-specific label taxonomies | Available |
| [Getting Started with DLP](./dlp.md) | DLP policies for Exchange, SharePoint/OneDrive, Teams, and endpoints | Coming soon |

## Why start with MIP?

MIP is the foundation. Labels define what data is sensitive — DLP policies then enforce rules based on those labels. Deploy labels first, then layer DLP on top.
