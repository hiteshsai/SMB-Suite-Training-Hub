# Project Guidelines

## Overview

Docusaurus 3.10.1 docs-only site for Microsoft SMB partner enablement — covering Microsoft 365 Business Premium, Defender Suite add-ons, and Purview Suite for Business Premium. Deployed to GitHub Pages at `microsoft/SMB-Suite-Training-Hub`.

See [README.md](README.md) for content map and [CONTRIBUTING.md](CONTRIBUTING.md) for the contribution workflow.

## Build and Test

All commands run from the `website/` directory. Requires Node 20+.

```bash
npm install          # install dependencies
npm run start        # dev server with hot reload
npm run build        # production build — run before submitting changes
npm run clear        # clear Docusaurus cache if builds behave unexpectedly
```

`npm run build` is the validation gate — it will throw on broken links and warn on broken markdown links (configured in `docusaurus.config.js`).

## Architecture

- **Docusaurus classic preset**, docs-only (blog disabled), docs served at root (`/`).
- **Sidebar is autogenerated** from `_category_.json` files — no manual sidebar config. Every new directory needs a `_category_.json`.
- **Config**: `website/docusaurus.config.js` (site metadata, navbar, footer, broken-link policy).
- **Styling**: `website/src/css/custom.css` — Microsoft blue theme (`#106ebe`).
- **Static assets**: `website/static/img/` — currently empty, place images here.

## Content Conventions

### Directory structure

| Section | Position | Purpose |
| ------- | -------- | ------- |
| Licensing | 2 | Business Premium, Defender, Purview packaging |
| Purview | 3 | Compliance and data protection training |
| Partner Guidance | 4 | Conversation flow and workshop guidance |
| Assets | 5 | NPF phase-1 asset catalog and source links |
| Scenarios | 6 | Customer scenarios / POC-in-a-box modules |

### Creating a new doc page

1. Add a `.md` file with this frontmatter:

   ```yaml
   ---
   sidebar_position: <number>
   title: <Page Title>
   ---
   ```

2. If adding a new directory, include a `_category_.json`:

   ```json
   {
     "label": "Section Name",
     "position": <number>,
     "collapsed": false
   }
   ```

### Naming

- **Directories**: PascalCase (`Purview`, `Partner-Guidance`)
- **Files**: kebab-case (`business-premium.md`, `poc-plan.md`)
- **Sidebar ordering**: Use `sidebar_position` in frontmatter and `position` in `_category_.json`

### Content patterns

- Lead with a partner-facing summary, then structure with tables and bullet lists.
- Link to Microsoft Learn docs rather than duplicating official content.
- Use relative markdown links between docs (e.g., `[MIP features](./MIP/features.md)`).
- Asset catalog entries should include: owner, status, audience, format, source URL, and last-reviewed date.
